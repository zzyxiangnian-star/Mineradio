# Windows 外壳、蜂窝播放与 DeepSeek 兼容修复设计

日期：2026-07-06

## 目标

修复以下四个已确认问题，同时保留现有播放器、托盘菜单和 AI DJ 的其他功能：

1. 托盘右键菜单底部的“退出”按钮必须始终可见且可用。
2. Windows 任务栏悬停预览应以当前歌曲专辑封面为主体，原生按钮应显示上一首、播放/暂停、下一首图标。
3. 点击右侧蜂窝唱片后立即播放，但唱片及蜂窝视口不得因点击或播放状态同步而移动。
4. AI DJ 使用 DeepSeek 官方接口时，应自动采用正确的认证、模型和请求参数。

## 根因

- 托盘菜单窗口固定为 420px 高，而实际内容高度超过窗口，且页面禁止溢出，导致“退出”被裁掉。
- 任务栏按钮复用了应用图标，未提供播放控制专用图标；任务栏缩略图仍是主窗口默认快照。
- 蜂窝点击和播放状态同步都调用 `selectIndex()`，该函数会计算中心吸附位置并执行位移动画。
- 当前保存的 DeepSeek 配置仍使用 MiMo 默认模型 `mimo-v2.5-pro`、`api-key` 请求头和 `max_completion_tokens` 字段；DeepSeek 官方接口需要 Bearer 认证和 `max_tokens`。

## 设计

### 托盘菜单

- 根据菜单实际内容提高默认窗口高度。
- 将退出操作放在固定底部操作区。
- 当屏幕工作区高度不足时限制窗口高度，并允许中间菜单区滚动。
- 保留原有 `quit` IPC 命令和真正退出应用的行为。

### Windows 任务栏预览

- Renderer 继续使用现有桌面外壳状态上报当前歌曲标题、歌手和封面。
- 主进程维护当前封面对应的任务栏预览状态，并通过 Windows/Electron 支持的缩略图裁剪能力显示专辑封面预览区域。
- 封面为空或加载失败时回退到 Mineradio 默认图标，不影响任务栏预览可用性。
- 上一首、播放/暂停、下一首分别使用独立透明图标；播放状态变化时只替换中间图标。
- 非 Windows 平台跳过缩略图和 Thumbar 增强，保持现有行为。

### 蜂窝唱片交互

- 指针点击唱片只调用播放适配器，不调用 `selectIndex()` 或 `animateToOffset()`。
- `syncTrack()` 只更新 `playingIndex` 并重绘播放高亮，不主动居中唱片。
- 拖拽、惯性和拖拽结束后的吸附行为保持不变；本次只移除“点击播放导致移动”。
- 键盘或程序触发的唱片点击遵循相同规则。

### DeepSeek 兼容

- 根据 Base URL 主机名识别 DeepSeek 官方接口。
- DeepSeek 配置自动使用 `Authorization: Bearer`。
- 当 DeepSeek 配置仍为空或仍为 MiMo 默认模型时，使用 `deepseek-v4-pro`；用户明确填写其他 DeepSeek 模型时予以保留。
- DeepSeek 请求使用 `max_tokens`，并继续使用 `/chat/completions` 路径。
- MiMo 继续使用现有 `api-key`、默认模型和请求字段，不产生回归。
- 设置页读取配置时展示最终生效的模型和认证方式，连接测试返回可理解的错误信息。

## 测试与验收

- 自动测试验证退出项存在且布局不会依赖被裁剪区域。
- 自动测试验证任务栏三个图标不再复用应用图标，并验证缩略图封面更新路径。
- 自动测试验证点击与 `syncTrack()` 不再调用蜂窝居中吸附。
- 自动测试覆盖 DeepSeek Base URL、Bearer、默认模型、`max_tokens` 与 MiMo 兼容路径。
- 运行 JavaScript 语法检查、现有模块测试和 `git diff --check`。
- Windows 实机启动后检查托盘菜单、任务栏悬停预览、蜂窝点击播放和 DeepSeek 连接测试。

## 非目标

- 不重写播放器队列、拖拽惯性或播放内核。
- 不引入 C++、Native Addon 或第三方 Windows 外壳扩展。
- 不改动用户已有的有效 API Key。
