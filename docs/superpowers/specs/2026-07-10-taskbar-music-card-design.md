# Mineradio Windows 任务栏音乐卡片设计

日期：2026-07-10

## 背景

Mineradio 当前通过 Electron 的 `BrowserWindow.setThumbnailClip()` 裁切主窗口中央区域，并使用 `setThumbarButtons()` 提供上一首、播放/暂停、下一首按钮。Windows 任务栏缩略图因此仍然来自主窗口最终合成画面，表现为播放器或视频视觉截图。

现有页面虽然包含 `#taskbar-preview-card`，但该节点设置了 `opacity: 0`、负层级和禁止指针事件，不参与最终可见合成。Electron 公开 API 只能裁切窗口内容，不能直接把任意音乐卡片位图设置为 DWM 任务栏缩略图。

本设计在不修改音频播放、解码、播放列表核心和音频管理逻辑的前提下，增加一个仅面向 Windows 任务栏 UI 的原生桥接层，使任务栏悬停缩略图显示真实歌曲信息和专辑封面的音乐卡片。

## 目标

- Windows 任务栏悬停缩略图不再显示主播放器或视频视觉截图。
- 缩略图表现为轻量、现代、接近 QQ 音乐的音乐控制卡片。
- 顶部显示 Mineradio Logo、当前歌曲标题和歌手名称。
- 中部显示当前歌曲 metadata 中的高清正方形专辑封面，并进行圆角裁切。
- 底部在视觉上衔接 Windows 原生上一首、播放/暂停、下一首按钮。
- 歌曲、封面和播放状态变化时自动更新。
- 原生按钮复用现有播放器命令，点击后立即响应。
- 原生模块失败时不影响播放器启动和播放，并自动降级。

## 非目标

- 不修改音频播放、解码、播放队列或音频管理核心逻辑。
- 不改变主播放器页面的整体视觉和交互。
- 不在缩略图位图内部实现可点击 HTML 控件；点击能力由 Windows 原生 Thumbar 按钮提供。
- 不要求非 Windows 平台支持相同的任务栏缩略图能力。
- 不以独立第二任务栏窗口或额外任务栏图标模拟主窗口缩略图。

## 技术方案

### 1. 总体架构

采用 Windows N-API 原生模块，在 Electron 主进程内为主窗口安装安全的窗口子类过程，响应 DWM 的自定义缩略图请求。

数据流如下：

1. Renderer 从现有播放状态生成任务栏状态。
2. Renderer 使用 Canvas 渲染一张 640×768 的高清音乐卡片。
3. 卡片以 PNG 或可验证的像素数据通过安全 IPC 发送到主进程。
4. 主进程使用 Electron `nativeImage` 解码并转换为原生模块接受的 BGRA 像素。
5. 原生模块缓存最新主位图，通知 DWM 缩略图已失效。
6. Windows 请求指定尺寸缩略图时，原生模块缩放缓存位图并调用 `DwmSetIconicThumbnail()`。
7. 原生 Thumbar 按钮继续通过现有 IPC 命令调用播放器控制函数。

### 2. 原生桥接职责

原生模块只负责 Windows 缩略图生命周期：

- 接收主窗口 HWND。
- 启用 `DWMWA_FORCE_ICONIC_REPRESENTATION` 和 `DWMWA_HAS_ICONIC_BITMAP`。
- 安装并在退出时移除窗口子类过程。
- 响应 `WM_DWMSENDICONICTHUMBNAIL`。
- 缓存、缩放和释放 BGRA/DIB 位图。
- 调用 `DwmSetIconicThumbnail()` 和 `DwmInvalidateIconicBitmaps()`。
- 在窗口销毁、模块卸载和应用退出时恢复窗口状态并释放 GDI 资源。

原生模块不得：

- 读取播放器状态。
- 控制音频或播放队列。
- 创建第二个任务栏窗口。
- 把异常传播到播放器核心启动流程。

原生模块提供最小接口：

- `attach(hwndBuffer)`：绑定主窗口。
- `updateBitmap(buffer, width, height)`：替换缓存的 BGRA 位图并请求刷新。
- `clearBitmap()`：清空自定义缩略图并准备降级。
- `detach()`：恢复窗口过程并释放资源。

所有接口返回明确结果或错误码。主进程统一捕获异常并执行降级。

### 3. Renderer 状态来源

继续复用现有状态，不创建第二套播放器状态：

- 当前歌曲：`playQueue[currentIdx]`，本地歌曲回退到 `currentLocalSong`。
- 标题：`song.name || song.title`。
- 歌手：`song.artist || song.ar || song.author`。
- 封面：`songCoverSrc(song, targetSize)`，优先自定义封面，再使用歌曲 metadata 封面。
- 播放状态：`audio.src && !audio.paused && !audio.ended`。
- 队列状态：`playQueue.length` 和 `hasTrack`。

状态同步触发点：

- 页面首次加载。
- 歌曲 UI 切换完成。
- 新封面加载成功或失败。
- `play`、`playing`、`pause`、`ended`、`emptied`、`abort`、`error`。
- 清空队列或切换到本地歌曲。
- 自定义封面发生变化。

状态使用内容键去重；普通同步保留短防抖，播放/暂停按钮状态和外壳命令响应走立即刷新，避免点击后的可见延迟。

### 4. 卡片渲染

Renderer 新增独立、无交互的任务栏卡片渲染器。它不依赖主舞台截图，也不读取视频画面。

主画布尺寸为 640×768，按 Windows 请求尺寸在原生层等比缩放。布局为：

- 顶部 104px：真实应用 Logo、单行歌曲标题、单行歌手名称。
- 中部：448×448 正方形封面，使用适度圆角和轻阴影。
- 底部：延续卡片背景和留白，让紧邻缩略图下方的三个原生 Thumbar 按钮在视觉上成为卡片控制区；位图内不重复绘制不可点击的假按钮。

视觉规则：

- 浅色浮动卡片、圆角、紧凑留白。
- 标题和歌手分别渲染，超长内容按可用宽度截断并加省略号。
- 封面始终使用歌曲 metadata 或用户自定义封面，不使用窗口、视频或播放器截图。
- 无当前歌曲时显示 Mineradio 品牌空闲卡片。
- 当前歌曲存在但封面加载失败时使用应用级音乐占位图，并保留歌曲标题和歌手。
- 应用 Logo 来自现有品牌资源，不使用 `MR` 文本代替。
- 原生 Thumbar 中间按钮在播放时显示暂停图形，在暂停时显示播放图形。

### 5. 封面加载与刷新动画

- 请求不低于 640×640 的封面资源；保留内联、自定义和本地封面支持。
- 网络封面通过现有同源 `/api/cover` 代理加载，避免 Canvas 污染和跨域失败。
- 歌曲切换时先更新标题和歌手，同时暂时保留上一张有效封面。
- 新封面解码成功后，用 5 至 6 帧、约 180ms 的低频交叉淡入生成卡片位图。
- 每一帧都校验歌曲切换 token；旧歌曲的延迟加载结果不得覆盖新歌曲。
- 动画仅在歌曲封面变化时运行，播放/暂停只重绘控制图标，不重复解码封面。
- DWM 刷新失败或系统处于高负载时允许直接切换最终帧，不阻塞播放线程。

### 6. 控制按钮

交互继续使用 `BrowserWindow.setThumbarButtons()`，因为 DWM 缩略图位图本身不接收 HTML 点击。

按钮行为：

- 上一首：发送 `prev`，Renderer 调用现有 `prevTrack()`。
- 播放/暂停：发送 `togglePlay`，Renderer 调用现有 `togglePlay()`。
- 下一首：发送 `next`，Renderer 调用现有 `nextTrack()`。

按钮样式使用透明背景的简洁单色图标，让 Windows 提供原生 hover、按下和禁用反馈。播放状态变化时重建中间按钮图标。无可播放歌曲时三个按钮禁用。

外壳命令收到后立即读取并上报一次状态；最终状态仍以音频元素事件为准，避免乐观状态与真实播放失败不一致。

## 文件与模块边界

预计修改：

- `public/index.html`：整理任务栏状态同步入口，接入独立卡片渲染器。
- `src/desktop/main.js`：加载原生桥接、更新位图、管理生命周期和降级。
- `src/desktop/preload.js`：增加受限的卡片位图 IPC 方法；保留现有命令桥接。
- `src/desktop/taskbarPreview.js`：扩展纯状态归一化和尺寸策略。
- `package.json`、打包配置：加入原生模块构建、Electron ABI 重建和 ASAR 解包规则。

预计新增：

- `public/taskbar-card-renderer.js`：在受限 Renderer 环境运行的卡片布局、文本截断和帧生成逻辑。
- `native/taskbar-thumbnail/`：N-API/Win32 原生模块。
- 卡片渲染、状态同步、IPC 和原生模块边界测试。

不会修改：

- 音频解码器。
- 音频 URL、音质选择和播放源逻辑。
- `playQueueAt()` 的核心播放流程。
- 播放列表排序、循环和随机核心算法。
- 服务端音乐供应商解析逻辑。

## 线程与资源安全

- 窗口子类安装、位图替换和移除均在 Electron 主线程发起。
- 原生模块对缓存位图使用明确所有权；替换前释放旧 GDI 对象。
- 处理 DWM 消息时只做必要缩放和提交，不调用 JavaScript 回调。
- HWND 销毁后所有更新接口返回不可用状态，不访问失效句柄。
- 应用 `before-quit`、窗口 `closed` 和模块清理钩子都执行幂等 `detach()`。
- 位图尺寸和缓冲区长度必须验证，拒绝异常尺寸及长度不匹配的数据。

## 降级策略

以下任一情况触发降级：

- 非 Windows 平台。
- 原生 `.node` 模块不存在或 ABI 不匹配。
- DWM 合成不可用。
- HWND 绑定失败。
- 位图解码或原生更新失败。

降级行为：

- 清除自定义缩略图属性，恢复 Windows 系统窗口缩略图。
- 保留窗口标题、任务栏 tooltip 和三个原生 Thumbar 控制按钮。
- 仅记录一次结构化警告，避免播放期间反复刷日志。
- 不弹阻断式错误，不影响播放器核心功能。

## 测试计划

### 自动化测试

- 状态归一化：标题、歌手、封面、playing、hasTrack。
- 无歌曲、网络封面失败和本地封面场景。
- 文本截断与画布尺寸。
- 封面 token 防止旧结果覆盖新歌曲。
- 播放/暂停图标映射。
- IPC 输入尺寸和缓冲区校验。
- 原生模块 attach/update/clear/detach 的幂等和错误路径。
- 非 Windows 或模块缺失时的降级路径。

### Chrome Renderer 测试

使用 Chrome 控制插件打开本地页面，测试：

- 播放真实歌曲后卡片预览包含正确标题、歌手和专辑封面。
- 不读取 Canvas/WebGL 视频舞台截图。
- 上一首、播放/暂停、下一首命令调用正确。
- 歌曲切换后标题、歌手和封面自动刷新。
- 播放和暂停状态实时同步。
- 封面请求无 404、CORS 或资源解码错误。
- Console 无新增错误。

Chrome 只能验证 Renderer 和数据逻辑，不能验证 Windows 原生任务栏。

### Electron Windows 实测

- 启动实际 Electron 应用并播放真实歌曲。
- 悬停任务栏图标，确认显示音乐卡片而非窗口或视频截图。
- 确认标题、歌手、Logo 和高清圆角专辑封面正确。
- 点击上一首、播放/暂停、下一首，确认立即生效。
- 确认播放/暂停按钮图标与真实音频状态一致。
- 连续切换歌曲，确认旧封面不会覆盖新封面。
- 最小化、恢复、隐藏到托盘和重新显示后重复检查。
- 在 100%、125%、150% DPI 下确认缩略图比例和清晰度。
- 检查主进程、Renderer 和原生模块日志无新增错误。

### 构建验证

- 原生模块按 Electron 33 ABI 成功构建。
- 开发运行可以加载 `.node` 文件。
- NSIS 与便携版产物包含解包后的原生模块。
- 安装版启动后执行一次完整任务栏验收。

只有 Chrome Renderer 测试、Electron Windows 实测和打包验证全部通过，才能认为任务完成。测试通过前不提交实现改动。

## 验收标准

- 任务栏缩略图不包含主窗口、视频画面或播放器截图。
- 显示真实应用 Logo、当前歌曲标题和歌手。
- 显示当前歌曲 metadata 的高清正方形圆角封面。
- 上一首、播放/暂停、下一首按钮可见、可用并有系统 hover 反馈。
- 播放/暂停图标与真实音频状态同步。
- 歌曲切换后所有数据自动刷新，无明显闪烁或旧数据覆盖。
- 原生桥接失败时播放器仍可正常启动和播放。
- Chrome、Electron Windows 和打包环境均无新增错误。
