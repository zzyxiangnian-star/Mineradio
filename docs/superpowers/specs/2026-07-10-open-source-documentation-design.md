# Mineradio 1.4.0 开源文档重写设计

## 目标

为 `zzyxiangnian-star/Mineradio` 重新建立一套中文为主、适合公开开源仓库的项目文档。文档既要帮助普通用户理解 Mineradio，也要让开发者能够从源码启动、测试、构建并参与贡献。

## 读者

- 想了解或体验 Mineradio 的普通用户。
- 想从源码运行、调试或构建 Windows 版本的开发者。
- 想提交 Issue、Pull Request 或安全问题的贡献者。

## 写作原则

- 中文为主，保留 Electron、Node.js、API、PR 等必要英文术语。
- 所有版本号、命令、环境变量、脚本和目录均以 1.4.0 源码为准。
- 不承诺源码中不存在或未经验证的功能。
- 清楚说明原项目地址以及当前仓库的功能迭代关系。
- 不在文档中放置 Cookie、API Key 或其他敏感信息。
- 优先使用可复制执行的命令和短段落，避免宣传性空话。

## 文档范围

### README.md

采用“产品展示 + 开发者上手”的阅读顺序：

1. 项目名称、简介、版本和许可证徽章。
2. 项目定位、原项目归属和当前维护说明。
3. Mineradio 1.4.0 的核心能力与版本亮点。
4. 现有截图和实际运行效果。
5. Electron、渲染层、本地服务和外部音乐服务之间的架构说明。
6. 环境要求、克隆、安装依赖、可选配置和启动命令。
7. 测试、原生模块重建、NSIS 安装包和便携版构建方法。
8. 精简且真实的项目目录说明。
9. 环境变量、本地数据、第三方服务和内容版权说明。
10. 常见问题、贡献入口、更新日志、安全政策和 MIT 许可证链接。

### CHANGELOG.md

- 使用清晰的版本分段记录 1.4.0、1.3.5 和 1.3.3。
- 1.4.0 覆盖任务栏音乐卡片、外部歌单导入、歌单面板、酷狗匹配、聊天壁纸和在线更新改进。
- 版本链接指向对应标签或比较页面，不使用会随时间漂移的错误链接。

### CONTRIBUTING.md

- 说明支持的 Node.js、npm 和 Windows 开发环境。
- 给出依赖安装、原生模块重建、启动和测试命令。
- 规定 Issue 信息、分支范围、提交说明和 PR 验证要求。
- 强调不得提交密钥、Cookie、用户数据、缓存、安装产物和第三方受限内容。
- 说明音乐平台接口可能受登录、地区、版权和会员状态影响。

### .env.example

- 保留可直接复制的变量名和安全的默认值。
- 为 AI 配置和用户数据目录添加简洁中文注释。
- 明确真实密钥只能写入本地 `.env`，不得提交仓库。

### SECURITY.md

- 声明当前支持的安全更新版本为 1.4.x。
- 给出私下报告漏洞的推荐方式，并要求避免在公开 Issue 中披露密钥或可利用细节。
- 列出报告应包含的影响范围、复现步骤、环境和建议修复信息。
- 说明第三方音乐平台故障、版权限制和账号权限通常不属于安全漏洞。

### LICENSE

- 保留现有 MIT 许可证正文，不为版本展示而改写许可证。
- 项目来源与维护关系放在 README 中说明。

## 内容核验

写作前从以下位置核验事实：

- `package.json` 和 `package-lock.json`：版本、脚本和依赖。
- `server.js`：本地服务、更新和音乐平台能力。
- `src/desktop/`：Electron 窗口、任务栏、壁纸和 preload 能力。
- `public/index.html`：用户可见功能和配置入口。
- `test/`：能够公开承诺的可测试行为。
- `build/`：Windows 构建和安装器配置。

## 验证标准

- 文档不存在 `TBD`、`TODO` 或占位文本。
- README 中的版本统一为 1.4.0。
- README 和 CONTRIBUTING 中的 npm 命令必须存在于 `package.json`。
- `.env.example` 的变量与源码读取逻辑一致。
- Markdown 链接使用有效的仓库相对路径或明确的外部地址。
- `npm test` 通过；文档修改不改变应用源码行为。
- Git diff 不包含密钥、Cookie、用户数据或构建产物。

## 非目标

- 不重写 `docs/superpowers/` 中已有的历史设计和实施计划。
- 不修改应用功能、界面、依赖版本或构建行为。
- 不把已安装的 Electron 运行库和可执行文件提交到源码树。
- 不在本次工作中创建 GitHub Release 或上传安装包附件。
