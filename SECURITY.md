# 安全政策

感谢你帮助维护 Mineradio 及其用户的安全。请负责任地报告安全问题，并在修复完成前避免公开可利用细节。

## 支持版本

| 版本 | 安全更新 |
|---|---|
| 1.4.x | ✅ 当前支持 |
| 1.3.x 及更早版本 | ❌ 不再提供安全更新 |

请先在最新的 `1.4.x` 源码或发布版本中确认问题仍然存在。

## 报告安全漏洞

优先使用仓库的 [Private vulnerability reporting](https://github.com/zzyxiangnian-star/Mineradio/security/advisories/new) 私下提交报告。如果该入口不可用，可以创建一个不包含漏洞细节的 Issue，请求维护者建立私下沟通渠道。

不要在公开 Issue、Discussion、Pull Request、日志或截图中披露：

- API Key、访问令牌、Cookie、账号凭据或其他密钥。
- 可直接利用的攻击载荷、完整利用步骤或尚未公开的绕过方式。
- 含有个人数据、用户目录、播放历史或账号信息的文件。

报告中建议包含：

- 受影响的 Mineradio 版本和提交。
- Windows、Node.js、Electron 及安装/源码运行环境。
- 漏洞类型、影响范围和现实攻击条件。
- 最小复现步骤或已脱敏的概念验证。
- 已观察到的结果与预期的安全边界。
- 可行的缓解措施或修复建议（如有）。

维护者会先确认收到报告，再评估影响、复现条件和修复范围。修复发布前，请与维护者协调公开时间；不要要求或承诺无法验证的固定响应时限。

## 安全问题范围

以下问题通常属于安全漏洞：

- 未经授权读取或写入本地敏感文件。
- 密钥、Cookie 或用户数据被意外发送、记录或暴露。
- Electron 主进程、preload 桥或本地服务中的权限绕过。
- 可由不可信输入触发的命令执行、路径穿越或持久化脚本注入。
- 更新包来源、校验或安装流程可被实际劫持。

以下情况通常不属于安全漏洞，但仍可作为普通 Issue 报告：

- 音乐平台接口暂时不可用或返回格式发生变化。
- 歌曲受版权、地区、VIP、登录状态或账号权限限制。
- 第三方服务限流、网络波动、镜像失效或歌曲地址过期。
- 需要用户主动修改源码、关闭系统安全能力或暴露自身凭据才能触发的问题。

## 敏感信息处理

- 不要把 `.env`、`.cookie`、`.qq-cookie`、`.kugou-cookie` 或用户数据目录提交到 Git。
- 怀疑密钥已经泄露时，应立即在对应服务撤销并轮换，不要只删除提交记录。
- 安全修复应尽量包含回归测试，但测试数据必须使用虚构凭据和脱敏路径。
- 第三方依赖问题请同时提供上游公告、受影响版本和 Mineradio 中的实际可达性证据。
